## ğŸŒ Web Interface Explanation

The project includes a simple and interactive **Streamlit web interface** to demonstrate the working of the trained models.

### Interface Components:
- **Text Area:**  
  Allows the user to paste a full programming problem description, similar to problems found on competitive programming platforms.

- **Predict Difficulty Button:**  
  Triggers the prediction process using the pre-trained models.

### Output Display:
Once the user clicks the predict button, the interface displays:
- **Predicted Difficulty Class:**  
  One of *Easy*, *Medium*, or *Hard*, generated by the classification model.
- **Predicted Difficulty Score:**  
  A numerical score representing the estimated difficulty level, generated by the regression model and normalized for better interpretability.

### Working Logic:
- The input text is converted into numerical features using the same **TF-IDF vectorizer** used during training.
- The classification model predicts the difficulty category.
- The regression model predicts the difficulty score.
- No retraining occurs during runtime; all predictions are performed using saved trained models.

The web interface allows users to easily test and visualize model predictions without interacting directly with the code.
